## Список ролей
- **Project Manager** – координирует работу команды и взаимодействие с бизнесом
- **Data Analyst** – исследует данные и формулирует бизнес-требования
- **Data Engineer** – создаёт пайплайны сбора и обработки данных
- **Data Scientist** – разрабатывает и улучшает ML-модели
- **MLOps Engineer** – обеспечивает инфраструктуру и производственное окружение моделей.
## Слоты для оценки 
- **experience_years** - опыт работы
- **hard_skills** - навыки
- **tools** - умение работать с инструментами
- **project_experience** - опыт работы в проектах
- **salary_expectation** - предполагаемая зарплата
- **education_level** - уровень образования
- **english_level** - уровень английского
- **availability** - когда сможет выйти на работу
- **work_format** - предпочитаемый формат работы
## Вес каждого критерия
| Критерий            | Слот                 | Максимальный балл | Логика                                                         |
| ------------------- | -------------------- | ----------------: | -------------------------------------------------------------- |
| Опыт работы         | `experience_years`   |                20 | Чем ближе опыт к требованию роли, тем выше балл                |
| Hard skills         | `hard_skills`        |                30 | Основной критерий соответствия роли                            |
| Инструменты         | `tools`              |                20 | Оценивается знание практических инструментов                   |
| Проектный опыт      | `project_experience` |                10 | Наличие опыта в учебных/рабочих проектах                       |
| Зарплатные ожидания | `salary_expectation` |                10 | Проверяется, укладывается ли кандидат в условный бюджет роли   |
| Образование         | `education_level`    |                 5 | Дополнительный плюс, но не главный критерий                    |
| Английский язык     | `english_level`      |                 5 | Дополнительный плюс, важен для документации и командной работы |

## Как считается балл по каждому критерию

Общая идея такая: у каждого слота есть максимальный вес, а кандидат получает весь вес или его часть.

Общая формула:

```
slot_score = max_slot_score * coefficient
```

Где:
- `slot_score` - балл за конкретный слот
- `max_slot_score` - максимальный вес критерия из таблицы выше
- `coefficient` - доля соответствия кандидата требованию роли от 0 до 1

### Опыт работы `experience_years`

Максимум: 20 баллов.

Если требование по опыту не задано, кандидат получает 20 баллов.

Если опыт кандидата больше или равен минимальному опыту роли:

```
experience_score = 20
```

Если опыт меньше требования:

```
experience_score = 20 * candidate_experience / required_experience
```

Пример: для роли нужно 1 год опыта, у кандидата 0.5 года:

```
experience_score = 20 * 0.5 / 1 = 10
```

Если опыт не указан, балл равен 0.

### Hard skills `hard_skills`

Максимум: 30 баллов.

Считается, сколько обязательных hard skills роли совпало с навыками кандидата.

```
hard_skills_score = 30 * matched_hard_skills_count / required_hard_skills_count
```

Пример: у роли 5 нужных навыков, у кандидата совпало 4:

```
hard_skills_score = 30 * 4 / 5 = 24
```

Совпадение считается не только по точному названию, но и по похожему тексту или синонимам, если они есть в базе требований.

### Инструменты `tools`

Максимум: 20 баллов.

Логика такая же, как у hard skills: считаем долю совпавших инструментов.

```
tools_score = 20 * matched_tools_count / required_tools_count
```

Пример: у роли 6 нужных инструментов, у кандидата совпало 3:

```
tools_score = 20 * 3 / 6 = 10
```

### Проектный опыт `project_experience`

Максимум: 10 баллов.

Если проектный опыт есть:

```
project_experience_score = 10
```

Если проектный опыт обязателен, но его нет:

```
project_experience_score = 0
```

Если проектный опыт только желателен, но его нет:

```
project_experience_score = 5
```

Если проектный опыт не требуется и не указан как желательный:

```
project_experience_score = 10
```

### Зарплатные ожидания `salary_expectation`

Максимум: 10 баллов.

Стартовое значение - 10

Если ожидания выше лимита, балл уменьшается пропорционально превышению.

Предельное значение - это лимит роли + 50%.

```
max_salary = salary_limit * 1.5
```

Если кандидат просит `max_salary` или больше:

```
salary_score = 0
```

Если кандидат просит больше лимита, но меньше `max_salary`, балл снижается линейно:

```
salary_range = max_salary - salary_limit
over_limit = candidate_salary - salary_limit
salary_score = 10 * (1 - over_limit / salary_range)
```

Пример: лимит 200000, кандидат хочет 220000:

```
max_salary = 200000 * 1.5 = 300000
salary_range = 300000 - 200000 = 100000
over_limit = 220000 - 200000 = 20000
salary_score = 10 * (1 - 20000 / 100000) = 8
```

Пример: лимит 200000, кандидат хочет 300000 или больше:

```
salary_score = 0
```

### Образование `education_level`

Максимум: 5 баллов.

Уровни образования сравниваются по порядку:

```
среднее < среднее специальное < неоконченное высшее < высшее
```

Если уровень кандидата равен требованию или выше:

```
education_score = 5
```

Если уровень кандидата на один ниже требования:

```
education_score = 2.5
```

Если уровень ниже больше чем на один или не указан:

```
education_score = 0
```

### Английский язык `english_level`

Максимум: 5 баллов.

Уровни английского сравниваются по порядку:

```
A0 < A1 < A2 < B1 < B2 < C1 < C2
```

Если уровень кандидата равен требованию или выше:

```
english_score = 5
```

Если уровень кандидата на один ниже требования:

```
english_score = 2.5
```

Если уровень ниже больше чем на один или не указан:

```
english_score = 0
```

Итоговая формула: screening_score = experience_score + hard_skills_score + tools_score + project_experience_score + salary_score + education_score + english_score

После суммирования применяются ограничители итогового балла для критичных условий:

| Условие                                              | Максимальный итоговый балл |
| ---------------------------------------------------- | -------------------------: |
| Опыт ниже минимума роли                              |                         74 |
| Совпало меньше 50% профильных hard skills            |                         74 |
| Совпало 20% или меньше профильных hard skills         |                         49 |
| Нет обязательного проектного опыта                   |                         49 |
| Зарплатные ожидания выше бюджета роли                |                         74 |
| Зарплатные ожидания выше бюджета роли на 50% и более |                         49 |

Это нужно, чтобы кандидат не получал `подходит` за счёт второстепенных критериев, если он не проходит обязательные требования роли.

## Пороговые значения `decision` 
|     Балл     | Значение `decision` |                   Интерпретация                   |
|:------------:|:-------------------:|:-------------------------------------------------:|
| от 75 до 100 |     `подходит`      |        Кандидат хорошо соответствует роли         |
| от 50 до 74  |      `резерв`       |    Кандидат частично подходит, но есть пробелы    |
|  от 0 до 49  |    `не подходит`    | Кандидат не соответствует минимальным требованиям |
Если лучшее значение score по всем ролям меньше 50, то recommended_role = "не определена", decision = "не подходит".
## Алгоритм выбора `recommended_role` 
1. Получаем заполненные слоты для кандидата 
   - target_role  
   - experience_years  
   - hard_skills  
   - tools  
   - project_experience  
   - salary_expectation  
   - education_level  
   - english_level  
   - availability  
   - work_format
2. Для каждой роли считаем `score`
3. Находим роль с максимальным `score`
Если максимальный `score` >= 50: `recommended_role` = роль с максимальным score.  
Если максимальный `score` < 50: `recommended_role` = "не определена".
4. Рассчитать `decision`:  
- `score` >= 75 - "подходит"  
- 50 <= `score` < 75 - "резерв"  
- `score` < 50 - "не подходит"
5. Сформировать `missing_requirements`: список требований, которых не хватает кандидату для `target_role`.
6. Если `target_role` отличается от `recommended_role`:  бот должен сообщить, что кандидат лучше подходит на другую роль.
## Список `missing_requirements` 
Данный список нужен, чтобы бот не просто отклонял заявку, а давал обратную связь.

|                   Условие                    |     Что добавить в `missing_requirements`      |
|:--------------------------------------------:|:----------------------------------------------:|
|      `experience_years` меньше минимума      |          `Недостаточно опыта работы`           |
|           Совпало мало hard skills           |     `Недостаточно профильных hard skills`      |
|              Совпало мало tools              |       `Не хватает ключевых инструментов`       |
| `project_experience = false` для DS/DE/MLOps |             `Нет проектного опыта`             |
|       `salary_expectation` выше лимита       |    `Зарплатные ожидания выше бюджета роли`     |
|         `english_level` ниже нужного         |      `Недостаточный уровень английского`       |
|        score по всем ролям меньше 50         | `Нет подходящей роли среди доступных вакансий` |
